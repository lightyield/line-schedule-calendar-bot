# line-schedule-calendar-bot

LINEで転送された複数のスケジュールメッセージ（スクールの連絡文など）を自動で集約し、Gemini APIを用いて指定フォーマットのマークダウン形式月別カレンダー（表）に変換して返信するLINE Botシステムです。

Google Apps Script (GAS) と Google スプレッドシートを組み合わせることで、**完全無料（サーバーレス・従量課金なし）**で最大2名（ご夫婦など）の環境で安定して動作します。

---

## 🛠 システム概要・アーキテクチャ

LINEの仕様上、複数メッセージを「一括転送」すると、システムには1通ずつバラバラに、ほぼ同時に届きます。これを正しく1つのカレンダーに統合するため、以下のバッファ（一時保存）機構を採用しています。

[LINE (複数メッセージを転送)] 
       │
       ▼ (ほぼ同時に連続リクエスト)
[GAS (Webhook受信)] 
       │
       ▼ (スプレッドシートの末尾にテキストを即時追記)
[Google スプレッドシート (一時保存)]
       │
       ▼ (最後の受信から約30秒〜1分間、追加がなければ発火)
[GAS (時限トリガー実行)]
       │
       ├─① スプレッドシートから全テキストを抽出・結合
       ├─② プロンプトを付与して Gemini API (Flashモデル) へ送信
       └─③ スプレッドシートのバッファをクリア
       │
       ▼ (AIがカレンダーを生成)
[Gemini API (Google AI Studio)]
       │
       ▼ (マークダウンの表形式テキスト)
[LINE (グループチャットへ返信)]

## ⚙ 動作要件・制限枠のクリア（完全無料）

- **LINE Messaging API**: プッシュメッセージ（Push Message）を使用するため、LINE公式アカウントの無料メッセージ通数枠（月200通）を消費します。少人数（個人・夫婦など）のスケジュール管理であれば、月200通で十分に賄うことができます。
- **Google Apps Script (GAS) / スプレッドシート**: トリガー実行時間およびデータ保存は無料枠の範囲内（1%未満の消費）。
- **Gemini API (Google AI Studio)**: Gemini 1.5 Flash または Gemini 2.0 Flash の無料ティア（1分間15リクエスト / 1日1500リクエスト）を利用。個人利用において上限に達することはありません。

## 🚀 導入手順（セットアップフロー）

詳細な手順は [docs/setup_guide.md](docs/setup_guide.md) に記載されています。大まかな流れは以下の通りです。

### 1. Google AI Studio での準備
1. Google AI Studio (https://aistudio.google.com/) にログイン。
2. **APIキー（API KEY）** を作成し、安全に手元に控える。

### 2. LINE Developers での準備
1. LINE Developers (https://developers.line.biz/) にログイン。
2. プロバイダーおよび「Messaging API」のチャネルを作成.
3. **チャネルアクセストークン（長期）** を発行。
4. **チャネルシークレット** を確認。
5. LINE公式アカウントの設定画面から「グループ・複数人チャットへの参加を許可」を**有効**にする。

### 3. Google スプレッドシート & GAS の設定
1. 新規スプレッドシートを作成し、シート名を buffer に変更。
2. メニューの「拡張機能」＞「Apps Script」を開く。
3. src/ 配下のコードを配置。
4. スクリプトプロパティ（環境変数）に以下を設定：
   - LINE_CHANNEL_ACCESS_TOKEN
   - GEMINI_API_KEY
5. GASを「ウェブアプリ」としてデプロイし、発行された Webhook URL をコピー。

### 4. 連携の仕上げ
1. LINE Developers の「Webhook URL」にGASのURLを貼り付け、「検証」が成功することを確認。
2. 「Webhookの利用」を**オン**にする。
3. 対象のLINE公式アカウント（Bot）を、夫婦2人が入っているLINEグループに招待する。

## 📝 コアロジック＆システムプロンプト

GASに組み込むシステムプロンプトの原本です。AIへの指示として固定で埋め込みます。

```markdown
# 依頼：スケジュールを月別カレンダー（表）に変換してください

以下の【スケジュール】を、指定の【ルール】に従って、マークダウン形式の月別カレンダー（表）に変換してください。

---

### 📅 基本ルール
1. **日曜日始まり**：曜日の見出しは、必ず【日 | 月 | 火 | 水 | 木 | 金 | 土】で固定してください。
2. **コンパクト表示**：カレンダーの表示期間は、スケジュール内の【最小の日（開始日）から最大の日（終了日）まで】に絞り、不要な週の行や空白期間はカットしてください（週の途中から始まってもOKです）。
3. **日付の網羅**：カレンダーの各マスには、必ず【日付】を漏れなく記載してください。

### 🟣・❌の表記ルール（※重要：対象月のみ適用）
1. **メイン対象月の判定**：「〇月〇月のスケジュール」とタイトルにあるような、その案内がメインとして扱っている月（以下、メイン対象月）のみ、すべての曜日で🟣❌の判定を行います。
2. **メイン対象月外の扱い**：メイン対象月以外の月（例：前後の月のレッスンが一部含まれる場合など）については、明示的に「休み」と記載のない限り、勘違いを防ぐため【❌】は記載せず、日付のみ（または🟣や⚠️のみ）をシンプルに記載してください。
3. **記号の基準**：メイン対象月において、レッスン（通常・コンクール等含む）の記載がある日は【🟣】、記載がない（休み）日は【❌】を日付の下に記載してください。
4. **シンプルの徹底**：曜日の文字（例：【月】）や、「○月○回目」といった回数表記は一切不要です。（例：「27<br>🟣」のようにシンプルにすること）
5. **クラスの統合**：「コンクールクラス」も通常レッスンと同じ扱いとし、表記は【🟣】に統一してください。

### ⚠️ 特別イベント・変更のルール
1. **カレンダー内の表記**：「時間変更」や「特別クラス（コンテンポラリーなど）」、「合宿」などの特殊な予定がある日は、カレンダー内には【⚠️】のアイコンのみを記載し、マスの幅を広げないようにしてください。
2. **表記の優先順位**：通常レッスンと特別予定（⚠️）が重複する日の場合、【⚠️】のみを優先して記載し、アイコンが1種類だけになるようにしてください（「🟣<br>⚠️」のように併記してはいけません）。
3. **補足の記載位置**：詳細な時間や場所、内容などの補足情報は、すべて【各月のカレンダーのすぐ下（月毎の欄外）】にまとめて記載してください。全体の最下部にまとめていけません。
```
