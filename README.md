# line-schedule-calendar-bot

LINEで転送された複数のスケジュールメッセージ（スクールの連絡文など）を自動で集約し、Gemini APIを用いて指定フォーマットのマークダウン形式月別カレンダー（表）に変換して返信するLINE Botシステムです。

Google Apps Script (GAS) と Google スプレッドシートを組み合わせることで、**完全無料（サーバーレス・従量課金なし）**で最大2名（ご夫婦など）の環境で安定して動作します。

---

## 🛠 システム概要・アーキテクチャ

LINEの仕様上、複数メッセージを「一括転送」すると、システムには1通ずつバラバラに、ほぼ同時に届きます。これを正しく1つのカレンダーに統合するため、以下のバッファ（一時保存）機構を採用しています。

[LINE (複数メッセージを転送)] 
       │
       ▼ (ほぼ同時に連続リクエスト)
[GAS (Webhook受信)] 
       │
       ▼ (スプレッドシートの末尾にテキストを即時追記)
[Google スプレッドシート (一時保存)]
       │
       ▼ (最後の受信から約30秒〜1分間、追加がなければ発火)
[GAS (時限トリガー実行)]
       │
       ├─① スプレッドシートから全テキストを抽出・結合
       ├─② プロンプトを付与して Gemini API (gemini-3.5-flash) へ送信
       │    ※ 503/429エラー時は最大5回リトライ（指数バックオフ）
       │    ※ 全リトライ失敗時はフォールバックモデル (gemini-2.5-flash) へ自動切替
       ├─③ Geminiが返したJSONを解析し、Flex Message (Carousel) を動的生成
       └─④ スプレッドシートのバッファをクリア
       │
       ▼ (AIがスケジュールデータをJSONで生成)
[Gemini API (Google AI Studio)]
       │
       ▼ (構造化JSON → Flex Message (Carousel) を動的生成)
[LINE (グループチャットへ返信：Flex Message)]

## ⚙ 動作要件・制限枠のクリア（完全無料）

- **LINE Messaging API**: プッシュメッセージ（Push Message）を使用するため、LINE公式アカウントの無料メッセージ通数枠（月200通）を消費します。少人数（個人・夫婦など）のスケジュール管理であれば、月200通で十分に賄うことができます。
- **Google Apps Script (GAS) / スプレッドシート**: トリガー実行時間およびデータ保存は無料枠の範囲内（1%未満の消費）。
- **Gemini API (Google AI Studio)**: Gemini 3.5 Flash（プライマリ）および Gemini 2.5 Flash（フォールバック）の無料ティアを利用。個人利用において上限に達することはありません。

## 🚀 導入手順（セットアップフロー）

詳細な手順は [docs/setup_guide.md](docs/setup_guide.md) に記載されています。大まかな流れは以下の通りです。

### 1. Google AI Studio での準備
1. Google AI Studio (https://aistudio.google.com/) にログイン。
2. **APIキー（API KEY）** を作成し、安全に手元に控える。

### 2. LINE Developers での準備
1. LINE Developers (https://developers.line.biz/) にログイン。
2. プロバイダーおよび「Messaging API」のチャネルを作成.
3. **チャネルアクセストークン（長期）** を発行。
4. **チャネルシークレット** を確認。
5. LINE公式アカウントの設定画面から「グループ・複数人チャットへの参加を許可」を**有効**にする。

### 3. Google スプレッドシート & GAS の設定
1. 新規スプレッドシートを作成し、シート名を buffer に変更。
2. メニューの「拡張機能」＞「Apps Script」を開く。
3. src/ 配下のコードを配置。
4. スクリプトプロパティ（環境変数）に以下を設定：
   - `LINE_CHANNEL_ACCESS_TOKEN`（必須）: LINEチャネルアクセストークン
   - `GEMINI_API_KEY`（必須）: Google AI Studio の APIキー
   - `GEMINI_MODEL`（任意）: 使用するGeminiモデル名（デフォルト: `gemini-3.5-flash`）
   - `GEMINI_FALLBACK_MODEL`（任意）: プライマリモデル障害時のフォールバックモデル名（デフォルト: `gemini-2.5-flash`）
5. GASを「ウェブアプリ」としてデプロイし、発行された Webhook URL をコピー。

### 4. 連携の仕上げ
1. LINE Developers の「Webhook URL」にGASのURLを貼り付け、「検証」が成功することを確認。
2. 「Webhookの利用」を**オン**にする。
3. 対象のLINE公式アカウント（Bot）を、夫婦2人が入っているLINEグループに招待する。

## 🔁 エラーハンドリング・耐障害性

Gemini APIの一時的なエラー（503: 高負荷、429: レート制限）に対して、以下の多層防御を実装しています。

| レイヤー | 内容 |
|---|---|
| **指数バックオフリトライ** | 最大5回リトライ（待機: 3秒 → 6秒 → 12秒 → 24秒 → 48秒） |
| **フォールバックモデル** | プライマリモデルの全リトライ失敗時、別モデルに自動切替して再試行 |
| **ユーザー通知** | 全モデルが失敗した場合のみ、LINEにエラーメッセージを送信 |

非一時的エラー（400: 不正リクエスト、401/403: 認証エラー等）はリトライせず即時エラー通知します。

## 📝 コアロジック＆システムプロンプト

GASに組み込むシステムプロンプトの原本です。Geminiへの指示として固定で埋め込みます。

```text
# 依頼：スケジュールを解析し、指定のJSONフォーマットで出力してください

以下の【スケジュール】から日付ごとの予定を月別に抽出し、指定の【JSONフォーマット】のみを出力してください。
マークダウンのコードブロック（```json ... ```）などの装飾は一切不要です。純粋なJSON文字列のみを返してください。

---

### 📅 抽出・判定ルール
1. スケジュール内に含まれる全ての月を判定し、"months" 配列に追加してください。
2. メイン対象月（案内がメインとして扱っている月）は "is_main": true を設定してください。**メイン対象月を必ず配列の先頭に置いてください。**
3. 【is_main: true の月】メイン対象月内の全ての日付について、レッスン（通常・コンクール等含む）の記載がある日は status を "🟣"、記載がない（休み）日は "❌" と判定してください。全日付を漏れなく列挙すること。
4. 【is_main: false の月】明示的なレッスン記載がある日は "🟣"、特殊予定（後述）がある日は "⚠️" のみ追加してください。❌は付けない。スケジュールに記載のない日はリストに含めないでください。
5. 「コンクールクラス」も通常レッスンと同じ扱いとし、status は "🟣" に統一してください。
6. 「時間変更」「特別クラス（コンテンポラリーなど）」「合宿」などの特殊な予定がある日は、status を "⚠️" とし、優先度を最優先にしてください（🟣や❌を上書きする）。その詳細（時間・内容・場所等）を "note" に記載してください。
7. 通常レッスン・休みの日の "note" は空文字（""）にしてください。

### 【JSONフォーマット】
{
  "months": [
    {
      "target_month": "YYYY-MM",
      "is_main": true,
      "events": [
        { "date": "YYYY-MM-DD", "status": "🟣", "note": "" },
        { "date": "YYYY-MM-DD", "status": "❌", "note": "" },
        { "date": "YYYY-MM-DD", "status": "⚠️", "note": "14:00〜 時間変更" }
      ]
    }
  ]
}
```

> Geminiの返答はGAS側で解析され、複数月の場合はLINEの **Flex Message (Carousel)** として送信されます。単一月の場合は単一バブルの Flex Message として送信されます。

## 🔄 CI/CD (自動デプロイ) の導入

本リポジトリは GitHub Actions と `@google/clasp` を用いた自動デプロイ環境に対応しています。
`main` ブランチへのプッシュまたはプルリクエストのマージ時に、GAS プロジェクトへ自動で最新コードが反映されます。

詳細な導入・設定手順については、**[GAS CI/CD 環境 構築ガイド](docs/cicd_setup.md)** を参照してください。
